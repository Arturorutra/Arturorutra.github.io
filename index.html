Linux Kernel / BootLoader / FS

发表于 2019-10-30 |

linux ( Ubuntu / Android / QT …)必须要挂载一个 FS（文件系统），但有些操作系统（如Vxworks / uC-os）就不需要。
嵌入式系统（Android）启动后，BootLoader先运行，检测内存，存储等设备，并设置好处理器等设备的时钟。设置完成后开始引导内核，也就是把zimage (内核)文件拷贝到内存中。拷贝完成后，跳进内核去执行。内核对外部设备进行驱动（蓝牙 / WIFI / SPI …）初始化，摸摸每个外设的脑袋，叫他们准备好，最后挂载文件系统，并执行文件系统的应用程序。执行一个个应用程序，安卓的界面就出来了。Android其实就是跑在Linux上的一个巨大应用程序。

BootLoader （引导程序）

在嵌入式操作系统中，BootLoader是在操作系统内核运行之前运行。可以初始化硬件设备、建立内存空间映射图，从而将系统的软硬件环境带到一个合适状态，以便为最终调用操作系统内核准备好正确的环境。在嵌入式系统中，通常并没有像BIOS那样的固件程序（注，有的嵌入式CPU也会内嵌一段短小的启动程序），因此整个系统的加载启动任务就完全由BootLoader来完成。在一个基于ARM7TDMI core的嵌入式系统中，系统在上电或复位时通常都从地址0x00000000处开始执行，而在这个地址处安排的通常就是系统的BootLoader程序。 —百度百科

U-Boot

U-Boot是最大的裸机程序，有协议栈，有文件系统。但是没有多任务运行机制，不能实现任务调度等操作系统应用的功能。

Linux Kernel （内核）

操作系统是一个用来和硬件打交道并为用户程序提供一个有限服务集的低级支撑软件。一个计算机系统是一个硬件和软件的共生体，它们互相依赖，不可分割。计算机的硬件，含有外围设备、处理器、内存、硬盘和其他的电子设备组成计算机的发动机。但是没有软件来操作和控制它，自身是不能工作的。完成这个控制工作的软件就称为操作系统，在Linux的术语中被称为“内核”，也可以称为“核心”。Linux内核的主要模块（或组件）分以下几个部分：存储管理、CPU和进程管理、文件系统、设备管理和驱动、网络通信，以及系统的初始化（引导）、系统调用等。 —百度百科

FS（文件系统）

文件系统是操作系统用于明确存储设备（常见的是磁盘，也有基于NAND Flash的固态硬盘）或分区上的文件的方法和数据结构；即在存储设备上组织文件的方法。操作系统中负责管理和存储文件信息的软件机构称为文件管理系统，简称文件系统。文件系统由三部分组成：文件系统的接口，对对象操纵和管理的软件集合，对象及属性。从系统角度来看，文件系统是对文件存储设备的空间进行组织和分配，负责文件存储并对存入的文件进行保护和检索的系统。具体地说，它负责为用户建立文件，存入、读出、修改、转储文件，控制文件的存取，当用户不再使用时撤销文件等。 —百度百科

加油站

U-Boot ，Kernel ，FS ，是嵌入式系统的基本概念，理解之后就明白了嵌入式系统的启动过程。为之后学习编译下载U-Boot，Kernel，FS打好基础。去者，往矣；然来者可期，今者，当一骑绝尘。 每天起床第一句，先给自己打个气 : Music《我怎么这么好看》！

一 、AP 与STATION

这里的 AP 与 STATION 是针对无线网络来描述的。AP模式用来发送信号，STATION模式用来接收信号。
AP全称Access Point，即接入点。类似于路由器产生的热点，手机用数据流量共享的热点，可以被其它多个由WIFI模块设备连接(手机，平板，笔记本电脑)。等同于有线网络的集线器HUB。
STATION就是连接AP的设备，就像是连接路由器无线网络的手机/笔记本/平板等。STATION不会生成在公共网络下产生可连接的WIFI名字，就像你的手机打开WIFI功能，是去扫描在公共网络中存在的AP，主动去连接。

这与TCP中的服务端(Server)于客户端(Clinet)是两个完全不同的概念。
AP的最大连接数是有限制的，这取决于AP的硬件性能。

二、你所看到的WIFI如何创建的?

WIFI是无线网络，是由板载|外延天线通过驱动函数向空气中发送 2.4G|5G 频段的电磁波。其中不同频段又区分不同的信道，不同信道的穿透能力以及抗干扰能力不同，详情可参考如下：

  1. WIFI信道频率对应
  2. 线通信频率分配表
  • WIFI模块初始化后，读取flash，加载系统配置，(默认AP模式)，调用板载天线驱动函数，通过特定的编码向空气中发送 2.4G/5G 的电磁波，作为AP等待被连接。
  • 手机/平板/笔记本等打开WIFI功能按钮时，加载系统默认设置，(默认STATION模式)，用天线驱动函数，接受默认信道中的 2.4G/5G 频段的电磁波，通过解码信息，在无线网络列表中显示解码后的电磁波信息，如:

手把手写加法器

发表于 2019-04-01 | 分类于 electric |

基于STC12C5A80S2开发板的一位加法器（状态机），单片机实现加法器，16键按键输入，输出结果通过数码管显示。欢迎大家共同交流。若有错误，请通过邮箱联系我。文末附上源码及流程图，欢迎大家检验

加法器 —> （A+B=C）

计算机是指输入、处理和输出的机器，单片机(MCU)当然可以运算，但单片机只又称为微型控制单元，做运算器的精度不会很高，博主抱着学习的心态来分享。让计算器输出结果 C ，自然要先输入数据 A（变量）、运算符号 + - * / 、B（变量），然后让计算器中的单片机计算出结果C，最后用数码管显示结果C。按键即输入，数码管显示即输出。

keyscan(按键扫描)——>输入

扫描函数这里用普通函数行列扫描，当然为了提升单片机效率可以使用中断函数扫描。通过按键扫描函数返回输入，16个按键对应 数字0-9 与符号 = ‘清零’ + - / 。在编程之前我们需要了解矩阵键盘的电路，常用16个按键（即4 4）要通过8个IO口来判断哪一个按键输入，4 * 4就是四行四列。这里检测按键的思路是逐列扫描，判断键盘是否被按下是通过什么呢？当然是通过检测IO口的高低电平了，先把行所连接的IO口设置为高电平，然后把列所连接的IO口分别设置为低电平（假设为P3^0-P3^3，P3 | =（0B00000111,0B00001011,0B00001101,0B00001110）），只有当某一列（若Y=1）为低电平且其它三列为高电平时按下按键，我们才能判断到底是第1（Y=1）列的第几（X）行被按下还是其它列的第几（X）行被按下。当按键被按下时，它所在的行和列就连在了一起，所在行的IO口高电平就拉低变为低电平，这是单片机检测到本来输出为高电平的IO口变为低电平，就认为按键被按下了，它的行数X就知道了，这一列的扫描就结束了，函数把这一按键（X，Y）所对应的编码值返回给主函数。主要步骤：

1. 把所有行所连接的IO口输出高电平。（也可设为低电平，只是检测按键被按下时电平相反）
2. 第Y（Y=1，2，3，4）列的IO口设置输出为低电平，其它三列的IO输出设置为1。（若第一列为检测到相应电平则程序继续检测下一列，即Y+1）
3. 通过程序来检测4（X）行（即4个IO口）是否为低电平。若为低电平则返回该按键对应的编码值，相反若为高电平则返回第二步，扫描下一列。

ATTENTION:当按键被按下时，它所在的行和列就连在了一起，所在行的IO口高电平就拉低变为低电平。

输入状态——>state

上面的三个步骤就是我们捕获输入返回给变量A的过程，下一个要输入的是运算符。如果对单片机不够了解的话可能会认为再按一下按键不就OK了吗？
因为按键被按下的时间中，单片机会一直返回这个值，变量A，运算符，变量B，符号‘=’都被你按的那一个键的返回值。本此使用单片机的机械周期为（1/11.0592M 秒），即每返回一个按键值大约需要9us（扫描函数指令按100条计算），以我目前的速度还差不少（0.0）。所以这其中涉及到一个状态检测的问题，要输出结果C，需要输入变量A，B，运算符和符号’=’,（单位加法器）也就是说要连续返回四次，即4个状态。假设初始状态state=0，按键返回值为keynum。

1. 当state=0时，才能把keynum赋值给A，赋值后state=1，同时数码管显示输入值，即A。
2. 当state=1时，才能把keynum赋值给运算符，赋值后state=2。
3. 当state=2时，才能把keynum赋值给B，赋值后state=3，同时数码管显示输入值，即B。
4. 当state=1时，才能把keynum赋值给等于符号，赋值后state=4，同时单片机处理运算，通过数码管显示结果。

输出——>

单片机首先判断运算符的类型（switch语句），然后计算得出结果，通过数码管显示驱动显示出来。这一部分比较简单，不做过多说明。
