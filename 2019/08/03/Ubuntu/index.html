Ubuntu16.04下的eclipse

发表于 2019-08-03 |

ubuntu16.04 安装eclipse失败 及 无法启动（已解决）

Install Failed

安装时每次启动 eclipse-inst 都失败，进度条卡住 install failed。笔者是个暴脾气的人，相信大力出奇迹，以为是JDK或JVM有问题，毕竟eclipse是需要JVM的支持，尝试了多次Delete和Install（当然删还是要删干净的），未果。根据提示定位到安装日志的第一个 ERROR: org.eclipse.equinox.p2.transport.ecf code=1002 Unable to read repository at http://download.eclipse.org/oomph/updates/milestone/latest/compositeContent.jar. (意思是说无法读取存储库，简称源不可用)

找度娘

众所周知，每个玩家都在经历无数玩家曾走过的路。二话不说，问度娘。果然，在2016年6月14日就有前辈在CSDN论坛[https://bbs.csdn.net/topics/391967001]发表了自己的问题，前辈们给出诸多解释。我现在整理一下：

  • 安装eclipse之前JDK要安装好 （显然我已安装好）
  • 网络环境不好或外网被和谐，可以直接下载jar包解决或者挂个vpn （网络环境是手机热点0.0，后证实否，外网被和谐，不大信！但是有同学成功了）
  • 安装路径设置有问题，路径名含中文字符，如下划线(_)等（这点我没有出现，也没有测试真实性，后辈们自己测）
  • 杀毒软件没关 （我尼玛是Ubuntu啊，我还没杀毒软件，软火墙都没开，Windows下我都没失败！）
  • 多安装几次就安装好了（我安装了不下10回！）

TIP：root权限在linux系统里的重要性不言而喻，不论是root还是管理员，又或者普通用户或是访客，这种严密的等级制度保障了linux系统的安全！不同的文件有不同的权限，所属不同的组，拥有者，不同用户的执行权限自然就不同了

是她是她就是她——eclipse

那日把她从Eclipse Foundation请到我家之后，她还带着一些行李，她说那是她的工作环境，必须带着，不然不能工作。我一笑而过，还好我的车够大，不然就放不下了。用我（管理员）的权限把她安顿在我新建的SoftWare大厦里，她是第一个住户，我给她的房门外贴上了专属的名字 eclipse。后来到下班时间，我微笑着和她说再见，替她关上了门，回到了我的大房子。

mkdir SoftWare                      //建大厦SoftWare
cd Software                         //进大厦
mkdir eclipse                       //给她安排一个房子，安顿她的“工作环境”
cd ../home                          //关上门，下班回家了

次日清晨，我找她去工作，进了SoftWare大厦，走到Eclispe门前，敲门之后，她竟然警告我，岂有此理，在我的大厦里还能这样嚣张，还好言辞没有太过锐利，她说：“The Eclipse executable launcher was unable to locate its companion shared library.（eclipse可执行启动程序找不到它的伴生共享库 ）”。这是我才想起来，这个大厦是人工智能，每一个人进出房间包括大厦都要有我的大管家OS的权限才能进出。我连忙问大管家，她的权限。这是我看到进出这个房子的用户是root，组也是root，她怎么可能出的来！我连忙道歉：“对不起啊，eclipse是我昨天太累，忘了给你房子的权限”。一边说一边就把权限给了她。（我的eclipse，运行时有一个java包放在了root目录下，已修改权限，这里就不贴了）

cd arturo/SoftWare                  // 进入大厦
./Eclipse/eclipse                   //走到Eclispe房间门口，叫她来工作
